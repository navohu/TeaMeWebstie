User acceptance testing

These use cases are designed to cover some functional scenarios of the application and are supposed to be tested them by the help of end-users.

See use cases

The above given use cases are executed by different end-users. So, this are the results of the user-acceptance testing:

Problem

For the first use case “Choose a team”, there was a problem first. So that a user couldn’t select a team from the list of the teams he/she is in.

Solution

It was because of the bug in the application code that the buttons were not answering to the user and were not connecting to the database. After fixing the error, it became possible to choose a team from the list and continue with the selected team.

Conclusion

Except the problem in the first test case, the other cases passed the test and there was no bug or difficulties experienced by the end-users. As the bugs in the first testing use case solved by us, all the use cases executed for testing and passed the tests.
